Getting started

Installation

NezhaSim ships three ROS packages and pulls several upstream simulators via vcstool. The whole build takes about 15 minutes on a typical desktop.

Requirements

ComponentVersionNotes
OSUbuntu 20.04 LTSFocal Fossa
ROSNoeticros-noetic-desktop-full
Gazebo11via gazebo_ros
Build toolscatkin_tools, python3-vcstoolcatkin build workflow
GPUoptionalOnly needed for high-resolution rendering; physics runs on CPU
Heads-up: NezhaSim targets ROS 1 / Gazebo Classic. The roadmap moves to ROS 2 + PX4, but the current release depends on the Noetic toolchain.

1 · Install prerequisites

# ROS Noetic (skip if already installed)
sudo apt update
sudo apt install ros-noetic-desktop-full

# Build & dependency tooling
sudo apt install python3-catkin-tools python3-vcstool python3-rosdep
sudo rosdep init && rosdep update

2 · Create a workspace & clone

mkdir -p ~/nezha_ws/src && cd ~/nezha_ws/src
git clone <your-fork-url> nezha_sim

3 · Pull third-party dependencies

The upstream simulators (RotorS, UUV Simulator, MAVLink, ASV-Wave, Husky) are not vendored in the repo — they are listed in nezha.repos and fetched with vcstool.

cd ~/nezha_ws/src
vcs import < nezha_sim/nezha.repos

# mavlink_sitl_gazebo has a submodule:
( cd mavlink_sitl_gazebo && git submodule update --init --recursive )

4 · Resolve system deps & build

cd ~/nezha_ws
rosdep install --from-paths src --ignore-src -r -y
catkin build
source devel/setup.bash
Tip: add source ~/nezha_ws/devel/setup.bash to your ~/.bashrc so every new shell sees the workspace.

5 · Verify the install

Launch a transmedium quadrotor in a lake. Gazebo should open with the vehicle floating at the surface; the topics below confirm the hydrodynamics and Mechanical-Transparency streams are live.

roslaunch nezha_gazebo nezha_mini_lake.launch

# in another sourced terminal — list the decoupled force streams (MT)
rostopic list | grep -E 'force|wrench|buoyan'
Build failing? The most common causes are a missing mavlink submodule (re-run step 3) or unresolved system deps (re-run rosdep install). Make sure every dependency in nezha.repos was cloned into src/.
